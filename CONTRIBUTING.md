# Contributing to StudyTrack

Thanks for helping improve StudyTrack.

## Before you open a pull request

- Run `flutter analyze`.
- Run `flutter test`.
- Keep secrets out of the repository and follow [SECURITY.md](SECURITY.md).
- Update docs when behavior changes.

## Suggested workflow

1. Create a branch for your change.
2. Make the smallest focused update that solves the problem.
3. Verify the app still builds and tests pass.
4. Open a pull request with a short summary of the change and any follow-up work.

## Code style

- Follow the existing Flutter and Dart patterns in the repo.
- Prefer clear names and small, testable changes.
- Keep generated files and large build artifacts out of version control.
